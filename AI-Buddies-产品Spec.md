# AI Buddies — 产品规格说明书（Product Spec）

> 版本 v1 · 2026-06-19 · 状态：定稿，供开发实现
> 配套文档：`AI-Buddies-技术Spec.md`（技术实现）、`AI-Buddies-方案设计.md`（背景与决策）、`AI-Buddies-UI-原型.html`（交互原型，本 spec 的视觉基准）、`usage_buddy.py`（核心逻辑的可执行参考）

---

## 1. 概述

AI Buddies 是一个面向个人重度用户的 **Claude Code + Codex 用量监控与优化教练**，覆盖 macOS（菜单栏 + 窗口）与 iOS（App + 小组件）。它读取两个 CLI 在本地写下的用量数据，统一展示 token、等效费用、额度窗口和按项目/模型的拆分，并基于使用模式给出"省钱、防限流、防浪费"的可执行建议。

**一句话价值主张**：不只是又一个用量表盘，而是一个带优化建议的、跨 Claude/Codex 的、能装进口袋的用量教练。

### 1.1 目标
- 一眼看清两端当前额度状态（5 小时 + 每周），避免突然被限流，也避免浪费没用完的额度。
- 量化"等效花了多少钱"，回答"订阅值不值"。
- 定位浪费源（贵模型用在小任务、缓存命中低、某项目吃掉大量额度）。
- 给出具体、可执行的优化建议，而非只是数字。
- 在手机上随时可查、临近上限时收到提醒。

### 1.2 非目标（v1 不做）
- 不做 App 内账号登录 / OAuth 抓取（违反 Anthropic 2026-02 的第三方订阅 OAuth 禁令，有封号风险）。
- 不做团队/多人聚合、计费对账、发票。
- 不修改 CLI 行为、不代发 prompt、不远程控制会话。
- 不自建云服务器（用 CloudKit 取代）。

### 1.3 设计原则
- **本地优先**：数据来自本地文件，离线可用，隐私默认不出本机（除非用户开启 iOS 同步）。
- **诚实标注数据可信度**：能拿到权威值就标"权威"，只能估算就标"估算"，不假装精确。
- **建议是一等公民**：每个界面都给建议位置。
- **一眼可读**：颜色编码状态（绿/琥珀/红），无需细看数字即可判断。

## 2. 用户与场景

**主用户画像**：同时重度使用 Claude Code 与 Codex 的开发者（即本项目所有者）。订阅档位可能是 Claude Max、Codex Pro。痛点：现有工具要么单端、要么只在桌面、要么只显示数字不给建议；想在一处看清两端，并在手机上随时掌握。

**核心场景**
1. **写代码途中瞥一眼菜单栏**：还剩多少 5 小时额度？要不要现在跑大任务。
2. **临近上限**：收到提醒，决定缓一缓还是切到另一端。
3. **周中复盘**：本周等效花了多少、哪个项目/模型最贵、有没有浪费。
4. **离开电脑时**：在手机/锁屏 widget 上看两端额度。
5. **调优**：照着建议把小任务从 Opus 切到 Sonnet、修缓存配置。

## 3. 数据来源与合规（产品前提）

| 数据 | 来源 | 可信度 |
|---|---|---|
| Claude token / 模型 / 项目 | `~/.claude/projects/**/*.jsonl` 的 `assistant.message.usage` | 精确（需去重） |
| Claude 5h / 每周 剩余% | 仅在 API 响应 HTTP header，本地无 | **估算**（用窗口内用量代理） |
| Codex token / 模型 | `~/.codex/sessions/**/rollout-*.jsonl` 的 `token_count.total_token_usage`（累计） | 精确（需差分、排除子代理重放） |
| Codex 5h / 每周 剩余% | 同文件的 `rate_limits.primary/secondary`（`used_percent`/`resets_in_seconds`） | **权威** |
| 费用（$） | token × 价格表（订阅用户为"等效 API 成本"） | 估算（Claude 价已核实，Codex 价近似） |

合规底线：只读本地文件；不使用订阅 OAuth；用户"登录"是指照常用两个 CLI 登录，App 读其产物。

## 5. 功能清单（按界面）

### 5.1 macOS 菜单栏
- **状态项标题**：默认显示两端 5h 额度 `C 72% · X 86%`，可在设置切换为 `今日 $`。≥70% 显示琥珀警示图标，≥90% 红色。
- **下拉面板**：
  - Claude 卡片：`估算` 标签 + 5h/每周环 + 今日等效 $；点击进入仪表盘 Claude 页。
  - Codex 卡片：`权威` 标签 + 5h/每周环（含重置倒计时）+ 今日等效 $；点击进入仪表盘 Codex 页。
  - 最重要的 1 条建议（按严重度取首条）。
  - 刷新按钮、设置齿轮（弹窗：菜单栏显示 %/$、刷新间隔）、"打开仪表盘"。

### 5.2 macOS 窗口仪表盘
导航 7 项：
- **概览**：4 个 KPI（本周等效 $、Claude 5h 剩余、Codex 每周剩余、本周 token）；每日等效费用堆叠柱图（7/14/30 天切换）；按模型条形；Top 项目条形；建议（前 4 条）。KPI 卡可点击跳转。
- **Claude**：5h/每周大环（标"估算"+ 说明官方%不可得）；本周等效 $ + 调用数；token 明细（输入/输出/缓存读/缓存写）；最近会话列表（可下钻）。
- **Codex**：5h/每周大环（标"权威"+ 重置倒计时）；本周等效 $（标"价格近似"）；token 明细；最近会话。
- **项目**：所有项目按等效 $ 排序的条形列表；点击任一项目下钻（费用、token、用到的模型、会话；返回按钮）。
- **模型**：按模型的 token + 等效 $；附一条"贵模型省钱"提示。
- **建议**：全部建议，带分类标签。
- **设置**：见 §8。

### 5.3 iOS App
- **首页**：`今日/本周` 切换；该区间合计等效 $；Claude 卡（估算 + 双环）、Codex 卡（权威 + 双环）；最重要 1 条建议。
- **趋势**：近 7 天堆叠柱 + 每日列表。
- **建议**：全部建议卡片。
- **设置**：套餐、通知开关、首页默认（今日/本周）、外观。

### 5.4 iOS 小组件
- **小号**：单端（默认 Codex）两环 5h/每周。
- **中号**：两端 5h 进度条 + 今日合计 $ + 1 条建议。
- **锁屏/Watch**：环形 complication，显示某端 5h%。
- 点击小组件打开 App。

## 6. 四大指标定义

### 6.1 费用（$）估算
等效 API 成本 = Σ(token 各类 × 单价)。Claude 单价已核实（Opus/Sonnet/Haiku，含缓存读 0.1×、缓存写 1.25×）；Codex 单价为可配置近似值。聚合维度：天 / 会话 / 项目 / 模型。顶部给"价值倍数" = 周等效 / 月付价（折算）。

### 6.2 额度窗口（5 小时 + 每周）
- Codex：直接读 `rate_limits` 的 `used_percent` 与 `resets_in_seconds`，显示精确剩余%与倒计时。
- Claude：以窗口（最近 5h / 7d）内的 token/等效$ 作为**用量代理**，明确标"估算"；显示距重置倒计时（按窗口边界推算）。
- 颜色阈值（按已用%）：<70% 绿、70–90% 琥珀、≥90% 红。

### 6.3 按项目 / 模型拆分
按项目目录、模型族分组，给 token、等效 $、占比；条形排序定位最耗者；项目可下钻。

### 6.4 建议 / Tips
见 §7。

## 7. 建议 / 教练引擎（核心差异化）

规则化、数据驱动。每条规则：触发条件 → 文案 → 严重度。阈值可配。

| 分类 | 触发条件 | 文案要点 | 严重度 |
|---|---|---|---|
| 防限流 | Codex 5h `used_percent` ≥ 80 | 还剩 X，约 N 后重置，先缓重任务 | danger |
| 防限流 | Codex 每周 `used_percent` ≥ 75 | 按当前速度可能提前耗尽，留给关键任务 | warn |
| 防闲置 | Codex 每周 `used_percent` < 25 | 额度充裕，可放心多用 | success |
| 省钱(模型) | Opus 调用中输出 < 400 tok 的占比 > 40%（且 Opus 调用 ≥ 10） | 小任务换 Sonnet/Haiku，省约 5× | warn |
| 省钱(缓存) | Claude 缓存读取占比 < 30%（且总输入相关 token > 20 万） | 固定 CLAUDE.md、复用上下文省输入 | warn |
| 做得好 | Claude 缓存读取占比 > 70% | 缓存命中良好 | success |
| 价值 | 设置了月付价且本周等效 > 0 | 本周等效 ≈ $X（折月≈$Y），约 Z× 价值 | info |
| 防过度 | Claude 最近 5h 用量 > 3× 平时每 5h 均值（且 > $1） | 短时用量偏高，注意别冲顶 5h 窗口 | warn |

逻辑实现以 `usage_buddy.py` 的 `build_tips()` 为准；新增规则在此表追加。

## 8. 设置项
- Claude 套餐：Pro / Max 5x / Max 20x；Codex 套餐：Plus / Pro。
- 月付价（用于价值倍数）。
- 菜单栏显示：额度% / 今日$。
- 刷新间隔：1 / 5 / 10 分钟。
- iOS 首页默认：今日 / 本周。
- iOS 同步项目名脱敏：关闭 / 哈希化。
- 外观：浅 / 深 / 跟随系统。
- 通知：接近上限提醒开关 + 阈值。

---

## 9. 通知
- 触发：任一端 5h 或每周 `used_percent` 跨过阈值（默认 85%）。
- 形式：本地通知（macOS / iOS）。文案含端、窗口、剩余、重置倒计时。
- 防打扰：同一窗口同一阈值仅提醒一次，重置后复位。

---

## 10. 边界情况
- 无任何数据文件 → 引导页，说明数据位置与 CLI 登录前提。
- 旧版 Codex 日志无 `token_count` → 跳过并提示该会话无 token 数据。
- Codex 子代理 rollout 重放父历史 → 识别并排除，避免高估（界面注明检测到 N 个）。
- Claude 无权威额度% → 全程标"估算"，不与官方页强行对齐。
- 时区 → 按本地时区聚合"今日/每日"。
- 价格变动 → 价格表可更新（见技术 spec 价格源）。

---

## 11. 隐私
- 默认全部本地，不联网（除 iOS 同步）。
- 同步只传聚合数字 + 项目名（可脱敏为哈希），不传代码/对话内容。
- CloudKit 私有库，绑定用户自己的 iCloud。

---

## 12. 验收标准
- 在真实数据上，Claude/Codex 的 token 与 ccusage 量级一致（去重正确、子代理不重复计）。
- Codex 5h/每周% 与 `codex` 的 `/status` 显示一致（权威同源）。
- 菜单栏在文件更新后 ≤ 刷新间隔内反映变化。
- 所有原型中的可点项在真机均可用并展示对应信息。
- 临近上限能触发通知且不重复打扰。
- 浅/深色、各 widget 尺寸显示正常。
